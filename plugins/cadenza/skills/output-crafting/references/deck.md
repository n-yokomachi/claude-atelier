# Output Crafting Reference: Deck (SpeakerDeck)

Finishing flow for conference deck output.

## Step B-1: Assemble the slide structure

```
- Title slide
- Self-intro (if needed)
- Agenda
- Background / problem statement
- Body (one chapter per sub-issue)
- Main message (standalone slide)
- Conclusion
- Appendix (Q&A backup)
```

## Step B-2: Enforce one slide, one message

- Does each slide title state the single message it carries?
- Is the text amount kept low (verbal narration is the supplement)?
- Visual primary, text secondary?

## Step B-3: Design chapter divider slides

Insert chapter dividers per sub-issue to show the audience's current position.

## Step B-4: Polish the decisive slides

The main-message slide and the slide for the most important verification result deserve extra polish:

- Font size / margins
- Emphasis treatment
- Visual hierarchy

## Step B-5: Prepare the appendix

Prepare backup slides for likely Q&A topics. Verification results cut from the main flow, deeper-dive diagrams, etc., go here.
